The different values of position

Visual samples of the effect of the different position values when used with offset values (left, right, top and bottom). In addition, a look at their corresponding logical offset values (still work in progress and may not be stable).

Follow along the CSS Positioned Layout Module Level 3 Editor's Draft for more information and updates, keeping in mind this edition is constantly changing at this point. Current browser implementations in Chrome and Firefox use the inset-block-* and inset-inline-* syntax, which reflects the editor's draft version of the specification as of 23 April 2020.

position: static

The default value of position for all boxes.

static

position: relative

The positioned (green) box has offset values of top: 2em and left: 2em, and thus is offset by 2em from the top and left of its normal position.

relative

position: absolute

The positioned (green) box is absolutely positioned with an offset value of right: 0 alone. This pushes the box to the right edge of the container but keeps its vertical position.

absolute

position: fixed

The positioned (green) box does its own thing because its containing block will always be the viewport. With offset values of right: 0 and top: 0 it will always be in the top right corner of the whole page.

fixed

position: sticky (top)

The positioned (green) box behaves like a relatively positioned box until its containing block has scrolled the normal flow top position “out-of-frame”.

The box will then maintain a top position relative to the visible area as content below it scrolls into view.

sticky (top)

position: sticky (bottom)

The positioned (green) box behaves like a relatively positioned box until its containing block has scrolled the normal flow bottom position “out-of-frame”.

The box will then maintain a bottom position relative to the visible area as content above it scrolls into view.

sticky (bottom)

position: sticky (left)

The positioned (green) box behaves like a relatively positioned box until its containing block has scrolled the normal flow left position “out-of-frame”.

The box will then maintain a left position relative to the visible area as content to its right scrolls into view.

sticky (left)

position: sticky (right)

The positioned (green) box behaves like a relatively positioned box until its containing block has scrolled the normal flow right position “out-of-frame”.

The box will then maintain a right position relative to the visible area as content to its left scrolls into view.

sticky (right)

writing-mode: horizontal-rb

The logical top of a container uses inset-block-start, while the logical bottom of a container uses inset-block-end. The logical left of a container uses inset-inline-start, while the logical right of a container uses inset-inline-end.

The property values in each of the coloured spans reflect what was written in the CSS Positioned Layout Module Level 3 as of 23 April 2020.
